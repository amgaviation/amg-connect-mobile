# Design Rules

- Dark mode is primary.
- Light mode must remain clean, white/off-white, and operational.
- Use AMG Accent Blue for active states and primary actions.
- Use thin borders and restrained card surfaces.
- Avoid oversized generic SaaS dashboards.
- Avoid consumer-airline booking language.
- Avoid fake operational metrics, fake aircraft data, or fake live maps.
- Keep mobile screens focused on real workflows: requests, aircraft, documents, messages, quotes/invoices, profile, and account status.
- Use native controls and icons where appropriate.
- Do not add random UI kits or heavy component frameworks.
