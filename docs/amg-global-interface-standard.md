# AMG Global Interface Standard

AMG interfaces should feel premium, minimal, modern, clean, precise, operational, and private-client appropriate.

## Color System

- Midnight Navy `#050B14` is the primary dark-mode background.
- Deep Blue `#07111F` is the primary card and elevated surface color.
- Accent Blue `#3B82F6` is reserved for primary actions, links, active navigation, and focus.
- Slate Gray `#9CA3AF` and Light Gray `#C0C7D1` support secondary text, borders, and quiet metadata.
- White `#FFFFFF` is used for primary text on dark surfaces.

No yellow, gold, playful pastels, random gradients, or generic SaaS palettes.

## Tone

Use professional aviation language. Avoid playful travel copy, fake urgency, marketing claims that imply acceptance, and unsupported operational promises.

## Layout

Prefer dark-mode aviation surfaces, thin borders, clear spacing, and simple hierarchy. Mobile screens should be scan-friendly, not card-heavy dashboards with fake data.

## Status Colors

Use green only for true success, amber only for true warning, and red only for destructive or error states.

## Brand Assets

Use the approved AMG logo only. The current foundation references `assets/logo/amg-logo-white.png`, downloaded from the public live website path `https://www.amgaviationgroup.com/images/logo-white.png`.

The MVP icon config uses a square padded placeholder generated from the approved white logo on AMG Midnight Navy. Splash config temporarily reuses the approved white logo. Replace them only with final approved AMG app icon and splash assets before store submission. Do not generate substitute logo or icon artwork.

## Accessibility

Primary actions should have clear text labels, form fields need visible labels, status/error text should be readable on dark surfaces, and protected screens must remain scrollable on small devices. Icon-only actions need explicit accessibility labels.
